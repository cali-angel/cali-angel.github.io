Lecture - Visual Studio Code for Web Developers

For this assignment, we were to watch and take notes on a lecture about the basics of using Visual Studio Code aka VSCode.

Overview

  • VS code UI
  • Left hand side is the view bar
  • Contains the explorer page, search view, integrated Git view, debugger view, and extension view. On the bottom is the status view, and the top is the application menu
  • Command palate has all of VS codes commands (cntrl+shift+p)
  • Zen mode focuses on just the file your working on, which removes all the other side bars and things.
  • Basic editing
  • Use indentation to make code easier to read
  • Can indent by right clicking and choosing format document, and then choosing a text and then choosing format selection.
  • Use ctrl+i to select lines, and keep typing that to add more lines
  • Ctrl+arrows up or down to change which lines you want to select
  • Use ctrl+shift+/
  • The integrated terminal
  • Can access it in the menus or ctrl+`
  • Can add a terminal with the plus sign, or delete with the trash can
  • Ctrl+arrow up or down to scroll the terminal
  • To clear the terminal use ctrl+k
  • Integrated source control
  • Git --version to see the version of Git you have
  • It will notify you in the source control viewer
  • You can see what changed in your file and can either switch through the changes or go back to making your editor full screen
  •  On the editor, a green bar is an added line, a blue line is when a line has changed, red triangle means lines were deleted.
  • In the git view, you can stage the changes.
  •  You can commit changes as well and press the arrow icon on top.
  • Perform other git options using the 3 dots next to the arrow
  • Configuring preferences
  • To go to settings, click file and click settings
  • The search bar is on top
  • On the left side editor, you can’t change it, but on the right side, it is changeable. In user settings, any changes made there will apply to all your files.
  • You can change settings for a specific language.
  • Setting keyboard shortcuts
  • Go to preferences and click keyboard shortcuts.
  • Chose the shortcut you want to change, and then paste it in the json file.
  • After you change it, save it and it should save properly.
  • Using extensions
  • Go to VScode marketplace on the browser
  • You can go to the extensions view and search for any extension
  • Managing extensions in the app is easy, just hit the gear to go to the settings. Also, using the 3 dots gives you a whole list of options to further manage extensions
  • Essential extensions
  • HTML 5 Snippets
  • jQuery code snippets
  • Beautify formats your code, and can be changed upon preference
  • Express is a web host server
  • Rest  client gets rest API’s
  • Working with snippets
  • Snippets are basically shortcuts for typing up longer code.
  • Usually showed on the completion list with a box with 3 dots
  • Extensions can provide snippets, but you can make your own
  • Under preferences, go to user snippets, and select a language to use it for
  • You give it a name, then write the code you want it to insert, and then write a description.
  • Color and icon themes
  • Preferences which will have the color/theme entry and a pop up list will have all that are available
  • You can install new ones from the extension marketplace
  • Another one is choosing icon theme
  • They basically just change the icons used. VS code has some themes pre-installed already but again if you want different ones, you can always get them at the marketplace.
  • Files, folders, and projects
  • VSCode doesn’t need special project files to work on projects, however if you have them, it knows how to interact with them.
  • Can preview files without opening too many tabs that make workspace messy
  • You can see where files are located by right clicking and and picking an option, or on the side bar, chosing to open the folder in the OS file explorer.
  • Side by side editing
  • To work on the same file, press ctrl+\  or click the split editor icon
  • To have a different file, go to the sidebar, right click the file and press open on the side
  • You can toggle the side editors vertically or horizontally
  • IntelliSense
  • As soon as you start typing, it already gives suggestions. You can go up and down the list with the up and down arrows
  • To dismiss the pop up, press escape
  • To trigger it, press ctrl+space key
  • You can turn off this in settings
  • Finding and replacing
  • You can search your document by highlighting a certain thing.
  • You can refine your search with the options it gives you.
  • You can also use the view bar to search for things
  • Editor features
  • Has hotExit which remembers what you wrote even when you don’t save it.
  • Has auto save
  • You can also auto format
  • Web code editing
  • Using the toggle comment can help you easily comment out sections of code. You can press ctrl+/. Just highlight the code you want to comment out.
  • You can task to open preview on the side
  • Emmit snippets are included
  • Navigating code
  • Press ctrl+g to go to a specific line
  • Ctrl+shift+o will take you to different symbols in the file
  • Ctrl+p shows the most recently opened files
