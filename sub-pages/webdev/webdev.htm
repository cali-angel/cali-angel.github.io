Lecture - Becoming a Web Developer: Full Stack vs Front End

For this assignment we looked at a lynda.com lecture and took notes about this topic.

Overview

Getting Started

  • Types of Web Devs
  • Web designers build web pages
  • Front end mainly works with clients, using HTML, CSS, and JavaScript
  • Back end works with servers and languages associated with that.
  • People who work with both are full stack
  • Most web devs are expected to have knowledge on both but specialize on one thing.
  • The role of HTML
  • Web devs need to understand the relationships that one web page has to the others
  • Information architecture: structural design of a websites navigation
  • Need to be masters in the markup language, or tags.
  • HTML builds the Document Object Model, which organizes all the files of a website. If DOM isn’t coded correctly, the whole website won’t work. So HTML is really an important language to master.
  • The roll of CSS
  • There are 2 aspects of specialization in design: User experience (UX) and user interface (UI)
  • CSS: cascade style sheet
  • 2 parts to CSS: 1) how the style sheet translates the DOM in visual form 2) cascading style rules, which determine the priority of where the styles are rendered on a page.
  • Learning css also means understanding motion and interaction in relation to graphic design
  • The role of JavaScript
  • It is the programming language behind the interactions on a webpage.
  • It can edit, or delete certain aspects of a page
  • Warning: since the language updates rapidly, some browsers don’t keep up, which causes programmers to debug, which can be a lengthy task.
  • Some write in different programming languages to use different techniques which will later be converted to javascript.
  • Back end essentials
  • Web devs need to understand how databases work and how the software searches the info needed to personalize the experience for each user.
  • Different companies choose what webstack tool will work for them. The most popular are PHP and mySQL. Try learning these first, and then look into others.
  • Servers
  • They manage the interaction between data and the way it is presented on the browser.
  • Often, it customizes pages based on the parameters set by the web devs.
  • Larger websites use software that manages scheduling of content. Back ends are in charge of creating/managing systems, while front end are more about the design and functionality.
  • Servers run on different OS’s so to be more hirable, get to know more than one but specialize in a specific language.
  • Back ends will deal with servers more, but it is good to understand what happens in servers.
  • Server side scripting
  • These are languages that run on servers and convert information for clients.
  • Different languages, but essentially do the same thing, so best way is to expose yourself to different ones and choose one to master.
  • PHP is the most popular and will make you more marketable.
  • Help in creating larger websites
  • Database languages
  • It's important to know how to access data and include it in server side layouts.
  • Standard to when it come to database languages: Relational database management system called Sequel (SQL) which creates tables for data
  • NoSQL has records with no structure, and is good for storing large documents.

Learning and Training

  • Train for the industry
  • 4 ways of learning web dev: instructor training, self study, project building, and learning which only happens by working with real clients.
  • The software changes so rapidly that instead of learning specific processes, focus on the basics and build from there.
  • Learn to read online docs, examine others code, and discover ideas which will make your work more efficient.
  • Traditional education
  • Those with college degrees earn more money than those without, and the skills they learn there give them a more wider foundation.
  • Some companies require that you have a degree.
  • It is not required though.
  • Coding bootcamps
  • These are fast paced learning courses that teach the basics
  • Tuition is cheaper, its shorter amount of time, and it has a placement rate at about 88%
  • Teach different skills and techniques than college
  • However they have a low acceptance rate and it is a full time thing.
  • If you are absolutely certain that web dev is what you want to be, than this is a good option.
  • Other learning options
  • The web has some great tools that can teach you, but it also has bad ones that have old and contradicting info
  • There are, however, online groups that have up to date info like slack.com and stack overflow
  • Books are better since they are reviewed by industry professionals and are more in depth.
  • Github is a good place to get others software and build something else or something better with it.
  • Find good and reliable places to learn the everchanging updates and techniques for web dev. Also know how to meet other developers.
  • Social learning opportunities
  • Its good to make connections and know other developers since that is the best way to get jobs.
  • Meetup.com is a good way to meet up with developers in your area.
  • Also, get involved and contribute in some way.
  • Overall, its just about getting out there and showing your talent and passion.
  • Core developer tools
  • 3 tools that all devs should have
  • Editors: 2 types
  • Integrated Development Environments (IDEs): they can be very opinionated and have an overwhelming amount of tools, so they are harder to master.
  • Text editors: they have a smaller set of built in tool, but have plug ins that expand the tool set.
  • Have a mastery of terminals
  • Command line interface (CLI)
  • Have a mastery on browsers
  • Other essential tools
  • Learn database management tools like phpmyadmin
  • Version control is where you store old versions of code. A popular system is called git where your old versions are called comits, and a site to share your comits is called github.
  • Build tools let you set up a work flows that handle common tasks.
  • Learn frameworks, libraries of prebuilt code that help you create sites faster.
  •  Example is bootstrap and jQuery.
  • Explore wordpress which is a content management system.
  • Core concepts
  • Making your website accessible to all, including the disabled is a good practice to have.
  • Having agile workflows when creating a project. What this entails is re-examining the plans of the project to see if anything needs to be updated to keep up with the ever changing criteria of development
  • Test driven development helps to test features while creating them. It helps to create software that is efficient and error free.
  • Business and leadership skills
  • Have communication skills, so listen and be open minded
  • Also, be able to present and defend your ideas by presenting to a larger audience
  • For developers, make sure to have a github account with your past work on it.
  • Be able to negotiate your salary

Getting a Job

  • Types of jobs
  • Most identify as full stack developers however, true full stacks are very rare
  • Salary as well as work/life time are major factors when it comes to choosing a job
  • Maximize compensation
  • If you work for a bigger company, you will get paid a bit more
  • More experienced devs can get more money than newer ones.
  • Education makes a difference in the amount of money you make
  • Land a job
  • The best ways to get a job is through a referal
  • Recruiters play a big part in the industry
  • Seek the opportunity directly by connecting with people who are associated with that business
