Overview

Web development foundations: web technologies part 1

  • Understanding how the web works
  • Internet is a giant network, whereas the web is a way to transmit information.
  • The web is a series of interactions between clients and servers
  • Clients are devices that request information
  • Servers are applications that send that information, or services
  • Working with clients and servers
  • Every time we click a link or type in a URL, we request information from servers
  • URL - Uniform resource locator
  • URL is sent to Domain Name Server, which converts it in IP, then sends it to host server.
  • Then the information is sent to the client, however, if it is a dynamic site, the information will be processed by the server and then the appropriate information is sent back.
  • DNS and why it matters
  •  Every website has an IP address, and a domain name for humans to use.
  • The DNS system is a spread out database.
  • When you look a site up, it checks the default ISP server, and if it can’t be found, then it checks the root name server, which then looks for the top level domain server appropriate for the site your looking up. If it still can’t be found, then the error DNS message will show up.
  • If site isn’t properly registered with DNS then no one will be able to find the site.
  • If there are any multiple names for the site, it must be registered to the same IP
  • For future reference, make sure you are aware of the ownership of the site is, since if you build a site and register it in your name, the person who bought the site would have difficulty managing it.
  • When you migrate a site, the local DNS servers won’t refresh their cache for 24 hrs. So do it in a time where few visitors are on the site.
  • Internet protocols
  • Protocols are simple languages that allow different computers running different software and hardware to interact
  • The internet uses a few protocols, named the internet protocol suite.
  • TCP/IP - transmission control protocol and internet protocol.
  • Internet protocol broken down to 4 layers
  • Application layer
  • Link layer
  • Transport layer
  • Internet layer
  • Exploring HTTP
  • Http - hypertext transfer protocol, which transfers information through the web
  • Other common protocols
  • FTP - file transfer
  • SMTP - simple mail transfer
  • RTP - real time transfer
  • Http stateless nature means that once a request is made to a server, it is then discarded.
  • Https is a protocol that is layered with a secure protocol
  • Anatomy of a URL
  • Its made up of the protocol, the subdomain, the domain, and the top level domain
  • We can specify what directory we want from typing in the URL
  • How browsers work
  • They basically send requests for information to servers, and then once receiving the data, they are able to render it on the screen.
  • Typical browser components
  • Browser UI
  • Internal engine
  • Rendering engine
  • Networking
  • Data storage
  • JavaScript interpreter
  • Things that differ from one browser to the next are the rendering engines
  • Rendering engines
  • Trident
  • Gecko
  • Presto
  • Webkit
  • Blink
  •  Web standards and the W3C
  • In the 90’s there was a period of time where no browsers were compatible with one another, so developers had to make sites specifically for each one.
  • This lead to the creation of W3C which at this point started to recommend web tech and strategies, but no one was listening.
  • They then started declaring standards and began educating developers, as well as the general public to use these standards
  • By learning these standards and keeping up with them, you can ensure that all you web pages will work on browsers.
  • Since they are always updating standards, make sure which standards are fully supported, semi-supported, or not supported at all.
  • Web server basics
  •  Servers are just computers with special sets of software
  • They can host multiple websites or one dedicated website
  • Sites are stored on files
  • Web Stacks are software that work together to build and process sites
  •  They consist of OS, web server, database server, and programming language
  • Its important to know what stacks that servers run so when you develop a site, you know what server to put it on.

Front end technology

  • Front end design
  • It is the UI layer of sites
  • The languages used are HTML, CSS, and JS
  • A lot more processing is happening within the browser itself that before, had to be done on a server.
  • Front end designers are more visual, than actual developers
  • Typical front end designer roles
  • Creating mockups and visual standards
  • Structuring semantic contend
  • Handling accessibility
  • Site typography
  • Layout
  • Design
  • Interactivity and animation
  • Site imagery
  • HTML, language of the web
  • Its an essential for all web designers
  • HTML 5 is the new standard
  • Structuring HTML
  • Html gives text structure
  • The very top is where the dog tag declaration goes
  • HTML is like a sandwich, all the tags are put in this order
  • Head tag is where all the extra stuff goes in like css files and other files needed to make the page look nice.
  • Controlling presentation through CSS
  • The cascade = CSS can be rendered in external files, embedded in HTML, or can be put inline with HTML tags.
  • Css are made up of 2 parts: the selector, and the declaration
  • Selector tells the browser what element to style
  • Declaration has the actual styling rules
  • The rules have 2 parts, property and the value

Web development foundations: web technologies part 2

  • Client-side scripting with JavaScript
  • JavaScript is a scripting language made by Netscape in 1995 to increase functionality of its browser.
  • It can allow you to create interactive elements
  • Can open windows and give feedback to users
  • Updates, sorts, and presents data in the browser
  • No server interaction is needed
  • JavaScript frameworks are already made commands that make it easier to use JS
  • Learn the basics
  • Then learn the frameworks
  • Common image types
  • File names include JPG, GIF, PNG, SVG
  • Its good to understand how images work in browsers
  • Find the balance between size and quality
  • Important how it plays a role on mobile devices
  • Only use graphic files that browsers support
  • Jpeg- Joint photographic experts group
  • Uses lossy compression to reduce file size
  • Not recommended to use jpeg as source file
  • GIF - graphics interchange format
  • It became the common file on the internet
  • 8 bit graphics, limited to 256 colors
  • Used mostly for icons/images of solid color
  • Support limited animations
  • PNG - portable network graphics
  • They support transparency and lossless compression
  • They support millions of colors
  • They are a larger file size
  • SVG - Scalable vector graphics
  • It can be scaled to any size without loss of resolution
  • Coded in SVG markup, which is like xml. Can modify them through code.
  • They are larger than compressed counterparts
  • What is an api?
  • API- application programming interface
  • Set of instructions and standards for accessing services or components of a specific system
  • Makes services publicly available for developers
  • Smaller sites can use larger web apps to help modify their site
  • They limit the amount of data or types of services they make available
  • Api’s change fast so make sure they are monitored so that they work properly
  • Sites go down sometimes, so make sure you have a plan
  • HTML 5 API’s
  • Allows designers to include lots of things without plugins
  • Api’s mature independently and this reduces the size of HTML 5 specs
  • Media api
  • Designed to control loading and playback of videos
  • Drag and drop
  • Allows to drag and drop function on page elements
  • Includes method of handling draging and dropping elements from outside the browser
  • App cache api
  • Stores data offline
  • Allows devs to create persistent applications that work even if there is no internet connection
  • Data can later be synced when there is a connection
  • Geolocation api
  • Standalone api associated with HTML 5
  • Allows developers to find current location of users and react to it\
  • Canvas api
  • Allows devs to draw within the browser
  • Comes in use with games and other interactions on the website
  • Web fonts
  • Used to use the system fonts only
  • Now it has changed with web fonts
  • It allows the downloading and temporary installation of fonts within the browser
  • It is used to describe the CSS @font-face technique
  • Font formats designed for web usage: EOT - Embedded OpenType, WOFF - Web Open Font Format
  • You can use font hosts like Typekit, which help you avoid license problems for one thing
  • You can host fonts yourself
  • Server-side scripting
  • The client sends a request to the server to have it process something. Then the server processes the information and sends it back to the client.
  • Scripts are hidden from the client
  • Processing done on the server can increase servers load, especially if it is a high traffic site, which causes the site to run slow.
  • Processing data usually requires a page refresh on the browsers
  •  AJAX and other client side bridges help enhance the usability of server-side scripting
  • The importance of PHP
  • It pairs with apache servers and mySQL
  • Php is embedded in HTML and then converted to HTML
  • You can find php in all hosting plans
  • Lots of developers in this language
  • Other popular server languages
  • JSP - Java Server Pages
  • Part of larger java Framework
  • Used mostly for enterprise level sites
  • Easy to find hosting, yet there are lots of different java platforms. Need to find the proper one
  • .NET is part of a larger framework
  • Used for large scale sites and is harder to use
  • Must be hosted on a windows server
  • Coldfusion is added to HTML by CFML, its own markup language
  • Central admin makes tasks simple
  • Sites that use this language often need less code than other languages
  • Hosts can be hard to find and more expensive
  • Python is an open source language and is very powerful
  • It has a variety of web related libraries that makes it easy to build sites and apps
  • Used to create large sites or apps
  • Ruby has been around for 20 years
  • Is a general purpose dev language
  • Gained popularity due to its Ruby on rails framework
  • It has a simple syntax and powerful feats, making a popular choice for dynamic sites
  • 3 main factors when choosing a server language
  • Platform cost
  • Developer network
  • Ease of use
  • Learn as much as you can about the languages
  • Dealing with data
  • It can be stored locally on the client, in a simple structured text doc, or a database
  • HTML 5 allows data to be stored on the client to help the user pick up where they left off
  • Also, files used for this are XML and JSON
  • Most efficient data storage is from databases
  • When using databases, all you need to do is update the info on the database and the rest is scripted on the website
  • There are 2 competing ways on keeping databases on the client side: indexed DB, and web SQL
  • There are 2 types of DB’s: relational and non-realational
  • Using SQL databases
  • It is the standard of managing and retrieving data from DB’s
  • It uses syntax that is simple and logical, very easy to learn
  • They can be refined to get a more exact search
  • Data retrieved from a DB is a dataset
  • 7 versions of SQL, all with different extensions
  • They are written in web application code
  • There are dedicated apps like mySQL that help create and manage DB’s
  • CMS - content management system
  • An app that helps manage, publish, and archive sites content
  • Updating content can be done through and CMS editor, making it easy for non devs or non tech people
  • Allows you to create user groups with various permission levels
  • They offer plugins and extensions that add powerful features to sites
  • Most are hosted online
  • Others are installed
  • Some limit the design you can do with the sites, and others give you a wide range of options of what you can do with it
  • CDN - content delivery networks
  • System that shares information over a series of servers
  •  Used largely to server static resources like JS, and multimedia
  • Some host single types of resources
  • Others serve as delivery and content aggregators like youtube
  • There are free and commercial CDNs
  • Using one adds a layer of complexity to your site
  •  Cloud services
  • Cloud computing uses a network of computers that can perform tasks by using the power and scale of the internet
  • These services are usually used through apps
  • What is GitHub
  • A distribution service that allows users to store their content online and can be version controlled and shared
  • It is built around Git
  • Allows you to manage projects, publish files and store revisions
  • Anyone can copy your work and either make their own things with it, or pull to have you except any changes they made on it
  • JavaScript libraries
  • Its as collection of prewritten JS code
  • They extend the language by adding functionality
  • Simply need to include the library on your file
  • It can extend your coding capabilities
  • It has been tested and it works
  • It increases the size of files
  • Using libraries while not knowing native JS limits your knowlegde of when to use libraries and when not to use them
  • Frameworks and boilerplates
  • They have css files that have predesigned rules, html structures, and JS
  • Boilerplate is set of templates that is built around a specific goal
  • Frameworks are assets for building sites
  • UI frameworks are fully functional dev tools. They add overhead but give developers the widest set of tools to work with
  • Minimal frameworks typically have a singular goal or focus
  • Help with rapid prototyping, but isn’t used for final products\
  • CSS Preprocessors
  • Its a scripting language that extends the functionality of CSS and is compiled into native CSS code before publishing
  • They add functionality that regular css lacks
  • The syntax makes writing css more efficient
  • Browser inconsistencies are dealt with automatically when the CSS is processed
  • Sass, Less, Stylus are examples
  • All need to know the preprocessor syntax
  • Processed css can be bloated an inefficient
  • Adds more complexity to work flow
